# Nodami 搭建教程

## 一. 准备工具

- **FinalShell 下载地址**: https://www.hostbuf.com/

## 二. 准备最少 1 台 LINUX 服务器

### 服务器选择注意事项

```text
1. 系统选择        推荐使用 Linux 系统，建议 Ubuntu 22 或以上版本。

2. 宽带要求        推荐带宽：50M 以上  
                        示例：观看 4K 视频至少需要 30M，为了保证快进不卡顿，建议 100M 以上。

3. 线路选择        节点延迟主要取决于线路类型。常见线路：国际 BGP、CN2、IEPL、大陆优化  
                        例如：在海外使用中国 IP 节点打游戏，建议选择 IEPL 线路，延迟最低。

4. 机房位置选择     尽量选择距离你实际使用地较近的机房位置以降低延迟。  
                        例如：人在香港，推荐选择深圳或广东的服务器，比北京低延迟。
```                       
### 我自己常用机房

### 5. 添加节点（等待服务器状态为在线）

#### 节点参数解释：

- **新增方式**
    - 批量搭建用：多个 IP 或多服务器情况
    - 一般情况选择默认即可

- **服务器**
    - 选择你搭建节点的目标服务器

- **内网穿透**
    - 一般用户忽略，主要用于拨号服务器场景

- **节点分组 / 节点地区**
    - 自定义标签，便于分类管理

- **出口类型**
    - 多 IP 服务器时使用，默认即可

- **协议选择**（推荐 Hysteria2）  
  | 协议        | 特点说明 |
  |-------------|----------|
  | **Shadowsocks** | 速度最快，但易被封锁 |
  | **ShadowTLS**   | 了解较少，自行研究 |
  | **Vmess / Vless / Trojan** | 速度适中，配合 gRPC 伪装抗封锁效果佳 |
  | **Hysteria2**    | 速度适中，抗封锁能力强 |
  | **Tuic**         | 延迟低、适合游戏，抗封锁能力未知 |

- **服务端口**
    - 留空系统将自动分配

- **中转协议**
    - 默认留空即可，适用于中转场景（如客户端 → 中转服务器 → 节点 → 网页）

## 6. 后台测速设置

- 需要单独配置一台测速服务器
- 若测速服务器与节点在同一机房，返回 `---` 或 `0sm`（延迟 < 1ms）

---

## 7. 多订阅节点配置

- 若希望不同订阅地址返回不同节点，  
  可在「自定义节点」中自行配置。

## 四. 客户端使用

> 面板首页推荐了多个客户端工具

我们以 **Hiddify** 为例：
